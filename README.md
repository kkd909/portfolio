## Design Philosophy

**Systems over one-offs.**  
Every component is part of a larger architecture. Tokens compose into semantics, semantics into components, components into patterns.

**Governance as enablement.**  
Rules should make the right choice the easy choice. Audit dashboards, auto-fix tools, and clear violation reports scale consistency without friction.

**Performance as a system property.**  
WCAG compliance, bundle optimization, and semantic HTML aren't bolted on — they're architected in from the start.
